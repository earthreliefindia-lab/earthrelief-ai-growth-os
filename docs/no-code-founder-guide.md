# No-Code Founder Guide

This guide explains Earth Relief AI Growth OS in founder-friendly language.

## What This System Does

The AI Growth OS helps Earth Relief create and manage growth ideas without giving AI unsafe control.

It can:

```text
Analyze trends
Create social media ideas
Suggest SEO improvements
Prepare content drafts
Store ideas for approval
Move approved posts to scheduling
```

It must not:

```text
Publish automatically
Edit website automatically
Deploy code automatically
Make unverified claims
Bypass founder approval
```

## Simple Mental Model

```text
AI Brain = thinks and writes
Automation = moves data between tools
Approval Queue = decision table
Postiz = social media scheduler
Supabase = database
Google Drive = business memory
```

## First Workflow To Use

Start with manual trend input.

Example:

```text
Trend: Hero defeats villain
Platform: Instagram
Goal: Promote sustainable packaging for businesses
```

The AI should return:

```text
Trend summary
Earth Relief adaptation
Reel idea
Caption
CTA
Risk
Confidence
Approval required
```

## Founder Decision Options

You can respond naturally:

```text
approve
reject
change CTA
rewrite for LinkedIn
make it more premium
make it more emotional
```

## Daily Operating Routine

1. Add new trend idea.
2. Review AI output.
3. Approve, reject, or modify.
4. Move approved content to Postiz draft.
5. Schedule manually after final review.

## Weekly SEO Routine

1. Run Search Console report.
2. Let SEO Agent find opportunities.
3. Review suggestions.
4. Approve safe changes.
5. Send approved items to website task list.

## Safety Checklist

Before approving any public content, check:

```text
Is the claim verified?
Is the message on-brand?
Is the CTA clear?
Is the risk acceptable?
Does this build long-term Earth Relief trust?
```
